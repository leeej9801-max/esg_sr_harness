# Core Principles

## 절대 유지 원칙
- Master = 정의
- Dictionary = 탐지
- Fact = 데이터
- Evidence = 근거
- AI = 생성

## 절대 금지
- Master와 Dictionary 혼합
- KPI를 issue_id에 직접 저장
- Evidence 없이 보고서 생성
- Impact / Financial 단일 점수화

## 1차 핵심 품질 축
1. KPI 정확성
2. Evidence 품질
3. Sub-topic 구조
4. Prompt 통제 설계

## 고도화 원칙
- Graph DB는 후순위 고도화
- Phase 4 전까지는 단일 하네스 기반 통제 우선
- 병렬 에이전트는 검증 단계에서만 제한적으로 사용
